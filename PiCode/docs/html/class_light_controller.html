◆ taskMethod()

void LightController::taskMethod ( )
virtual

This is the task method. The task method will be invoked periodically every period units of time.

This is the task method. It contains logic which is periodically invoked and executed.

1.0 Start by determining if there is anything event that has come in from the queue. If the queue has any item on it, dequeue it and process it. Continue this until the queue is empty.

1.1 If there are one or more items in the queue, process the first item by dequeuing it first.

1.2 Based on the commands in the queue, process it accordingly.

1.2.1 The On-off command has been received. The light should be turned off.

1.2.2 The on command has been received, we are to turn the light on.

1.2.3 A new PWM duty cylce has been received. Retrieve the duty cycle from the message and update within the class if the value is valid.

2.0 Determine whether the light is on or off based upon the mapOn variable. If it is off, do nothing, unless the button is pressed, in which case the logic should continue as if the lamp is on. If it is on, turn the LED on for a period of time, then go to sleep, then turn it back off.

2.1 The lamp is off. Do nothing and simply end this iteration of the periodic task.

2.2.1 Otherwise, the lamp should be on at current DC, so turn the lamp on by adjusting the appropriate GPIO pin.

2.2.2 Calculate how long to sleep such that the sleep period is the percentage of the duty cycle times the period.

2.2.3 Go to sleep for the given period.

2.2.4 Now turn the light off.

3.0 This ends the periodic task. It will execute again in 5ms if the task is setup properly.
