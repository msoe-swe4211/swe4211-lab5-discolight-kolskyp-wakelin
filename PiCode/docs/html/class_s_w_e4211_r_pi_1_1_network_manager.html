int NetworkManager::processReceivedMessage ( networkMessageStruct &  receivedMessage)
protected

This method will process a received network message. When it does this, it will calculate out and verify the checksum. If the checksum is correct, the contents will then be enqueued to the correct queue.

Parameters
receivedMessageThis is a reference to the received message structure.
Returns
The calculated checksum is returned.

1.0 Declare a variable for the calculated checksum with a value of 0.

2.0 If the security mode of the given message is 0, process it.

2.1.0 Convert the message to the appropriate endian format. Do this by converting each individual structure element accordingly and placing it back in the structure.

2.2.0 Calculate the XOR checksum of the data received, not including the checksum field.

2.3.0 Verify that the received checksum matchess the checksum that was transmitted.

2.3.1.0 The message is valid. Enqueue it to the right queue if the destination queue is valid and it is a COMMAND_MSG_TYPE.

3.0 Return the calculated checksum.

This is the run method for the class. It contains the code that is to run periodically on the given thread.

This is the run method for the class. It contains the code that represents the running thread.

1.0 Create a socket file descriptor. The socket is a tcp socket. If there is an error, indicate that the socket failed and return from the message.

2.0 Call setsockopt to setup the options on the socket. Make sure the options are set at the sopcket level, allow the reuse of local addresses, as well as ports. If there is an error, print out an error message and then return.

3.0 Setup the server which will run on this device so that the connection is to be using an IPv4 Internet protocols based format. The connection should also be allowed from any address.

4.0 Setup the server to listen on the desired port. The port must be converted to network endianess.

5.0 Bind the listening socket to the correct port. The bind will set the local port which will be used for connections from remote devices. Binding is a singleton action - only one process can be bound to a given port at a time. If there is a failure, print out an error and exit the program.

6.0 Loop so long as the thread is to continue running (i.e. a call to stop on this class has not occurred.)

6.1.0 Listen for a client that is trying to connect to this server. If there is an error, print out an error and exit.

6.2.0 Accept the connection from the client, setting the connected socket to the value returned from the accept call. If there is an error and the thread is to keep going, return after printing out an error if the thread is to continue running.

6.3.0 Report out the ip address of the client that has connected to this server using the appropriate method.

6.4.0 So long as the socket remains open and the thread continues running.

6.4.1.0 Create a networkMessageStruct that is to receive the given message.

6.4.2.0 Setup a char pointer to point to the start of the networkMessageStruct.

6.4.3.0 Receive the message into the buffer. Make sure to wait on all of the message being received (i.e. the full structure) When making the read, store the number of bytes received.

6.4.4.0 Check to see if 0 bytes were received, which would indicate a closed socket.

6.4.4.1.0 If we receive 0 bytes, the socket has been closed so abort and close the connected socket.

6.4.4.2.0 Process the received message.

6.5.0 Set the ip address of the connected client to be zero and broadcast this out as a new connected client.
